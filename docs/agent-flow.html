Agent System — How It Works

From Notion page → shipped full-stack app

Entry point
You (Main Claude session)
"Spin up @orchestrator-agent — here's my Notion page ID, start Phase 1"
↓spawns
Master coordinator
@orchestrator-agent
Reads Notion → writes PRD.md + UI_BRIEF.md locally → runs all phases → manages Notion kanban → never writes code
↓fetches from
Input source
Notion Page
PRD · UI Brief · Feature list
↓begins
Phase 1 — Documentation parallel + sequential
Run in parallel ↓
@pm-agent
→ docs/features.md
Features, user stories, build order
@system-design-agent
→ docs/system-design.md
Architecture, data flows
Then sequentially ↓
@db-agent
→ docs/schema.sql
PostgreSQL schema, indexes, triggers
@api-design-agent
→ docs/api-spec.md
Every route, request + response shape
All docs pushed to Notion as sub-pages + Kanban created
⏸ HUMAN GATE — review docs → reply "go ahead"
↓after approval
Phase 2 + Phase 3 run in parallel ↓
Phase 2 — Backend
One feature at a time, in order
@be-dev-agent
Builds routes for one feature
Hono + Bun + raw SQL
↓
@qa-agent
Tests all routes — happy path,
edge cases, auth, bad inputs
↓
@code-review-agent
Reviews git diff → APPROVED
or CHANGES REQUESTED
↺ QA fail or review changes → loops back to be-dev-agent
On approval → merge feature branch → next feature
Phase 3 — UI Mockups
Runs independently, doesn't block Phase 2
@ui-designer-agent
HTML mockups for every screen
4 states: default, loading, empty, error
→ docs/screens/*.html
↺ Human gives feedback → agent revises → repeat
⏸ HUMAN GATE — "screens approved"
↓when both Phase 2 + 3 are done
Phase 4 — Frontend screen by screen
Same loop as Phase 2, but for FE screens
@fe-dev-agent
Converts HTML mockup → real Next.js screen, wired to API
@qa-agent
Tests UI states, API wiring, error handling
@code-review-agent
Reviews diff → approve → merge
↺ Fail or changes → loops back to fe-dev-agent
↓all screens merged
App shipped to main ✓
All features built, tested, reviewed, merged · Notion kanban fully Done · build-log.md complete

agent-system · every agent globally available via @agent-name · orchestrated from a single Notion page
